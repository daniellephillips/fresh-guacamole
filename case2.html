Meadow Mountain

Device Management for IoT

February 2017

Product Overview

Meadow Mountain was the code name for a free cloud-based IDE built to support the rapid prototyping phase for IoT developer kits. Features included fully managed toolchains and a device manager for managing remote devices.

Role and Challenges

My role was design lead and UX product owner, engaging with various stakeholders across business groups. I managed the UX backlog and worked with the engineering team during sprints. The initial release called for MVP features that would accommodate IoT prototyping use cases.

The device manager was the value add feature which would eventually break off into its own service and product. Realizing and accounting for the security requirments presented the biggest challenge. From a usability perspective, a wrong configuration setting due to a confusing UI could make customers vulenrable so it was important to get it right. I worked egineering architects to understand device registration, security controls and data encryption, translated those findings to user requirements in our device registration flow.

Process and Key Contributions

  • Collaboration with key stakeholders and hardware SME’s to understand firmware flashing methods and security infrastructure for authenticating devices in the cloud.
  • Execution of complex interactive prototypes to simulate system interactions, and controls for device configuration.
  • Parternering with Human Factors Engineers to design and orchastrate a rigorous research and testing plan to validate non-functional requirements, end to end flow, and user understanding of visual metaphors and conceptual frameworks.
  • Synthesis, Evaluation, and Dissemination of Research Data to various teams.
